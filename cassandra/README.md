# cassandra

Three-node Apache Cassandra 5 cluster (one container per worker). CQL, JMX, and Prometheus metrics are exposed on maand-assigned ports. Admin credentials are generated at build time and stored in KV.

**Version:** 1.0.0

## Role in the stack

```
cassandra  --metrics (JMX exporter)-->  prometheus (scrape)
coroot_node_agent (on same workers)     host/container metrics only
```

Application clients connect to any node on `cassandra_cql_port` using the admin user from KV.

## Worker placement

| Setting | Value |
|---------|-------|
| Selector | `cassandra` |
| `deploy_parallel_count` | 1 (sequential bootstrap) |
| `update_parallel_count` | 1 (rolling restarts) |
| Memory | min `1024 mb`, max `4096 mb` (manifest) |

Workers must carry the `cassandra` label in `workers.json`. The first worker in catalog order (`cassandra_0` / `worker_0`) is the seed.

Example:

```json
{ "host": "10.48.200.1", "labels": ["cassandra", "worker"] }
```

## Configuration

### bucket.jobs.conf

No job-specific section required. Override memory if needed:

```toml
[cassandra]
memory = "4096 mb"
```

After editing: `maand build && maand deploy --jobs cassandra`.

### manifest.json — ports

| Port key | Use |
|----------|-----|
| `cassandra_cql_port` | Native CQL (`cqlsh`, drivers) |
| `cassandra_jmx_port` | JMX (nodetool, local tooling) |
| `cassandra_metrics_port` | Prometheus `/metrics` (JMX exporter javaagent) |
| `cassandra_storage_port` | Inter-node storage (plaintext; unused when `internode_encryption: all`) |
| `cassandra_ssl_storage_port` | Inter-node TLS (gossip, replication) |

All ports are allocated from the bucket range (`port_min`–`port_max` in `bucket.conf`).

### TLS certificates

| Cert | CN | Notes |
|------|-----|-------|
| `internode` | `cassandra` | PKCS#8; per-worker leaf cert for inter-node mTLS |

`entrypoint.sh` combines `internode.crt` + `internode.key` into `/var/lib/cassandra/tls/internode.pem` and copies `ca.crt` for the truststore. `cassandra.yaml` enables **`internode_encryption: all`** with `PEMBasedSslContextFactory` and `require_client_auth: true` (same pattern as zookeeper quorum TLS).

CQL remains on `cassandra_cql_port` with password auth only (no client TLS in this job).

### Secrets

Generated by `command_ensure_secrets` at **`post_build`** into `secrets/job/cassandra`:

| Key | Description |
|-----|-------------|
| `admin_username` | Superuser name (`cassandra`) |
| `admin_password` | Random url-safe token |

Rendered to `.env` on workers via `.env.tpl` (`getSecret`). Read with:

```bash
maand cat kv get secrets/job/cassandra admin_username --reveal
maand cat kv get secrets/job/cassandra admin_password --reveal
```

On first deploy, `command_y_init_admin` sets the password on the seed node after CQL is up.

### Key files

| File | Purpose |
|------|---------|
| `docker-compose.yml.tpl` | `cassandra:5`, host network, JMX exporter javaagent |
| `cassandra.yaml.tpl` | Cluster config (seeds, ports, auth, inter-node TLS) |
| `entrypoint.sh` | Build PEM keystore/truststore from maand certs |
| `.env.tpl` | Admin credentials for operators / compose |
| `jmx/jmx_exporter_config.yaml` | Whitelist-based JMX → Prometheus mapping |
| `Makefile` | Downloads javaagent jar; cert permissions; `start` / `stop` / `restart` |
| `_modules/command_wait_up.py` | Waits for `nodetool status` → UN after each node starts |
| `_modules/command_y_init_admin.py` | Sets admin password on seed (index 0) |
| `_modules/command_update_admin_password.py` | CLI: rotate admin password |
| `_modules/command_cluster_status.py` | CLI ring overview |

### Network

- **Host network** — container shares worker IP; no port mapping.
- Seed list is a single address (`cassandra_0`) to avoid split-brain on fresh clusters.
- **Inter-node TLS** — all gossip/replication on `cassandra_ssl_storage_port`; nodes present maand-issued `internode` certs.
- JMX exporter binds `0.0.0.0:<cassandra_metrics_port>` so Prometheus and maand health checks reach `/metrics`.

## Deploy

```bash
maand build
maand deploy --jobs cassandra
maand deploy --jobs prometheus   # pick up scrape config if _prometheus/ changed
```

Deploy is **sequential**: one node at a time, seed first. Each node must reach **UN** before the next starts.

### Health check

| Check | Target |
|-------|--------|
| TCP | `cassandra_cql_port` |
| HTTP | `cassandra_metrics_port` `/metrics` → 200 |

60 attempts × 5s timeout.

## Commands

| Command | When | Description |
|---------|------|-------------|
| `command_ensure_secrets` | post_build | Generate admin credentials (seed only) |
| `command_wait_up` | after_allocation_started | Block until node is UN |
| `command_y_init_admin` | after_allocation_started | Apply admin password on seed |
| `command_update_admin_password` | CLI | Rotate admin password in cluster and KV |
| `command_cluster_status` | CLI | Print `nodetool status` from seed |

### Cluster status

```bash
maand job_command cassandra command_cluster_status
```

Runs meaningful output from allocation index 0 only.

### Rotate admin password

```bash
maand job_command cassandra command_update_admin_password
```

Generates a new random password, applies it on the seed node via `ALTER ROLE`, updates `secrets/job/cassandra`, and refreshes `.env` on every cassandra worker. No redeploy required.

## Operations

### Connect with cqlsh

```bash
cqlsh <host> $(maand cat kv get maand/bucket cassandra_cql_port) \
  -u cassandra \
  -p "$(maand cat kv get secrets/job/cassandra admin_password --reveal)"
```

On a worker:

```bash
docker exec -it cassandra cqlsh 127.0.0.1 $(grep cassandra_cql_port ...) \
  -u cassandra -p "$(grep CASSANDRA_PASSWORD .env | cut -d= -f2-)"
```

### Verify metrics

```bash
curl -s http://<host>:$(maand cat kv get maand/bucket cassandra_metrics_port)/metrics | head
```

### Restart on a worker

```bash
ssh root@<worker-ip> 'make -C jobs/cassandra restart'
```

`make stop` runs `nodetool drain` before tearing down the container.

### Rolling upgrade

`update_parallel_count: 1` — maand restarts one node at a time. Expect brief client redirect/retry during each restart.

## Troubleshooting

| Symptom | Check |
|---------|-------|
| Node won't join / gossip errors | All nodes must have TLS enabled; verify `ss -nltp` shows `cassandra_ssl_storage_port`; `maand cat certs --jobs cassandra` |
| TLS handshake / certificate errors | Redeploy after cert rotation; check `./certs/internode.{crt,key}` and `ca.crt` on worker |
| Single-node cluster / split brain | Seed must be `cassandra_0` only; verify `CASSANDRA_SEEDS` in rendered `docker-compose.yml` |
| Health check fails, node UN | Metrics port must bind `0.0.0.0`; `curl http://<worker-ip>:<metrics_port>/metrics` |
| Admin init hangs | CQL port is not 9042 — init uses `cqlsh 127.0.0.1 <cassandra_cql_port>` |
| `ALTER ROLE` unauthorized | Default user cannot change own superuser flags; only password is set |
| Auth fails after enabling PasswordAuthenticator on old data | Fresh bootstrap may be required if cluster previously used `AllowAllAuthenticator` |
| Enabling TLS on existing plaintext cluster | Requires full sequential redeploy; all nodes must switch together — plan data backup |
| Missing javaagent jar | Run `make start` (downloads jar) or redeploy; jar lives in `jmx/` on worker |
| Prometheus target down | Worker reachable; port in KV; container running; firewall allows scrape from prometheus host |
